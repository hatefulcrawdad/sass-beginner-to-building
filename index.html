Sass?

Sass: Syntactically Awesome Style Sheets

Sass is the most mature, stable, and powerful professional grade CSS extension language in the world.

[ http://sass-lang.com ]

Sass is a CSS Preprocessor

  • Sass compiles into CSS. That’s all.
  • Sass solves a lot of problems that CSS has.
  • Only edit Sass files, never edit your CSS files!

Installing Sass

Ruby

  • gem install sass
  • sass --watch input.scss:output.css

Node

  • npm install node-sass
  • sass --watch input.scss:output.css

Sass Basics

Syntax

.sass

Write less using white space instead of curly brackets and semicolons.

.class
  property: value

.scss

Easiest transition from CSS, they kinda look similar.

.class {
  property: value;
}

Variables

Store values to use multiple times in your stylesheets.

.scss

// Variables
$font-stack-sans: 'Open Sans', Arial, Helvetica, sans-serif;
$brand-color: #5fcf80;

body {
  font-family: $font-stack-sans;
}

h1 {
  color: $brand-color;
}

.css

body {
  font-family: 'Open Sans', Arial, Helvetica, sans-serif;
}

h1 {
  color: #5fcf80;
}

Variable Abstractions

Reuse your default variables in a flexible way by creating “themes”.

.scss

// Default variables
$font-stack-sans: 'Open Sans', Arial, Helvetica, sans-serif;
$froggy: #5fcf80;

// Theme
$main--font: $font-stack-sans;
$main--color: $froggy;

body {
  font-family: $main--font;
}

h1 {
  color: $main--color;
}

.css

body {
  font-family: 'Open Sans', Arial, Helvetica, sans-serif;
}

h1 {
  color: #5fcf80;
}

Great for keeping things organized and usable without ever needing to change the core variables that have been setup.

Nesting

Organize your code into logical chunks.

.scss

.panel {
  padding: 20px;

  h2 {
    font-size: 20px;
  }
}

.css

.panel { padding: 20px; }
.panel h2 { font-size: 20px; }
  • Nested selector appends to the parent.
  • Never go more than 3 levels deep.
  • Nesting with an & is awesome!

Nesting: The Wrong Way

.scss

.panel {
  .sidebar {
    .copy {
      h2 {
        font-size: 20px;
      }
    }
  }
}

.css

.panel .sidebar .copy h2 { font-size: 20px; }

Nesting: The Right Way

.scss

$froggy: #5fcf80;

.panel-link {
  color: $froggy;

  &:hover,
  &:focus {
    color: darken($froggy, 20%)
  }
}

.css

.panel-link {
  color: #5fcf80;
}

.panel-link:hover, .panel-link:focus {
  color: #2e9a4e;
}

Nesting Properties

Save some writing by nesting namespaced properties.

.scss

$font-stack-sans: 'Open Sans', Arial, Helvetica, sans-serif;

p {
  font: {
    family: $font-stack-sans;
    size: 1em;
    weight: bold;
  }
}

.css

p {
  font-family: 'Open Sans', Arial, Helvetica, sans-serif;
  font-size: 1em;
  font-weight: bold;
}

Comments

Sass comes with 2 types of comments

Multiline Comments – /* */

  • Preserved in CSS when possible (not in compressed output).
  • If first letter is ! the comment will be preserved even in compressed output.
  • Can use variables and interpolation within multiline comments.

.scss

$version: “1.2.3”;
/* This is a multiline comment
 * that has CSS generated
 * by Awesomesauce version #{$version}. */

.css

/* This is a multiline comment
 * that has CSS generated
 * by Awesomesauce version 1.2.3. */

Single Line Comments – //

  • Will only exist in Sass files.
  • Can only be one line long.

.scss

// This is a single line comment,
// it will not show up in CSS output
body {
  background-color: #efe;
}

.css

body {
  background-color: #eeffee;
}

The Ampersand – &

Reference the parent selector while nesting.

.scss

a {
  property: value;
  &:hover {
    property: value;
  }
}

.css

a {
  property: value;
}
a:hover {
  property: value;
}

Nesting with Ampersands

  • Don’t just transverse through markup.
  • Used for pseudo selectors/elements.
  • Used for BEM style selectors.

Trailing Ampersands - .class &

Useful to keep styles grouped to an element but provide unique styling based on a parent selector.

Double Ampersands - & + &

Useful for adjacent selectors that share the same class name.

Getting Sassier

Extend Directive – @extend

  • One class needs all the styles of another class, plus styles of its own.
  • In our HTML, that means we always have to remember both classes
  • The @extend directive avoids this, like so:

.scss

.sidebar {
  background-color: #efe;
  border: solid 1px #ddd;
}

.sidebar-heavy {
  @extend .sidebar;
  border-width: 3px;
}

.css

.sidebar, .sidebar-heavy {
  background-color: #efe;
  border: solid 1px #ddd;
}

.sidebar-heavy {
  border-width: 3px;
}

Extend Variations

  • Use multiple extends within one selector.
  • Chain extends where one selectors extends the other and then a third selector extends the 2nd one.

.scss

.error {
  border: 1px #f00;
  background-color: #fdd;
}
.seriousError {
  @extend .error;
  border-width: 3px;
}
.criticalError {
  @extend .seriousError;
  position: fixed;
  top: 10%;
  bottom: 10%;
  left: 10%;
  right: 10%;
}

.css

.error, .seriousError, .criticalError {
  border: 1px #f00;
  background-color: #fdd;
}

.seriousError, .criticalError {
  border-width: 3px;
}

.criticalError {
  position: fixed;
  top: 10%;
  bottom: 10%;
  left: 10%;
  right: 10%;
}

Extend-only Selectors

  • I like to call these "Placeholder Selectors".
  • Sometimes you want a class this is only extended and not used in your HTML.
  • They use % instead of # or ..

.scss

%crazy-link {
  color: blue;
  font-weight: bold;
  font-size: 2em;
}

.main-link {
  @extend %crazy-link;
}

.secondary-link {
  @extend %crazy-link;
  background-color: #efe;
}

.css

.main-link, .secondary-link {
  color: blue;
  font-weight: bold;
  font-size: 2em;
}

.secondary-link {
  background-color: #efe;
}

Function Directive – @function

  • Return a value based on parameters you create.
  • Great for doing math calculations that will save you time

.scss

$grid-width: 40px;
$gutter-width: 10px;

@function grid-width($n) {
  @return $n * $grid-width + ($n - 1) * $gutter-width;
}

#sidebar { width: grid-width(5); }

.css

#sidebar {
  width: 240px;
}

Built in Functions

  • http://sass-lang.com/documentation/Sass/Script/Functions.html
  • RGB Functions – rgba($red, $green, $blue, $alpha)
  • HSL Functions – lightness($color)
  • Opacity Functions – opacify($color, $amount)
  • Color Functions – darken($color, $amount)
  • String Functions – str-length($string)
  • Number Functions – round($number)
  • List Functions – nth($list, $n)
  • Map Functions – map-get($map, $key)
  • Selector Functions – selector-nest($selectors…)
  • Introspection Functions – feature-exists($feature)

Mixin Directive

@mixin & @include

Simple Mixin

  • Store styles that you can repeat across different selectors in you Sass.
  • Different from @extend in that it repeats the styles and doesn’t group selectors in a list.
  • Use @include to declare the mixin on different selectors.

.scss

@mixin mixin-name {
  property: value;
}

div {
  @include mixin-name;
}

a {
  @include mixin-name;
}

.css

div {
  property: value;
}

a {
  property: value;
}

Mixins and @content

  • Create some styles and additionally be able to add on to them using @content within the mixin.

.scss

@mixin mixin-name {
  property: value;
  @content;
}

div {
  @include mixin-name {
    color: red;
  }
}

a {
  @include mixin-name {
    text-decoration: none;
  }
}

.css

div {
  property: value;
  color: red;
}

a {
  property: value;
  text-decoration: none;
}

Mixins with Arguments

  • Define certain parts of a mixin to accept arguments that you can customize each time you use a mixin.
  • Arguments can be false or null.

.scss

@mixin box($background: #fff, $border-radius: 4px) {
  background: $background;
  box-shadow: 0 2px 0 rgba(#000,.06);
  border-radius: $border-radius;
  position: relative;
}

.panel {
  @include box;

  &-sidebar {
    @include box(#efe, 8px);
  }
}

.css

.panel {
  background: #fff;
  box-shadow: 0 2px 0 rgba(#000,.06);
  border-radius: 4px;
  position: relative;
}

.panel-sidebar {
  background: #efe;
  box-shadow: 0 2px 0 rgba(#000,.06);
  border-radius: 8px;
  position: relative;
}

Mixins with Arguments Lists

  • Pass one or multiple arguments to a certain property within a mixin.

.scss

@mixin shadows($shadows...) {
  box-shadow: $shadows;
}

.box {
  @include shadow(0 0 1px rgba(#000, 0.15), 0 0 10px rgba(#000, 0.15));
}

.css

.box {
  box-shadow: 0 0 1px rgba(#000, 0.15), 0 0 10px rgba(#000, 0.15);
}

Advanced Goodies

Lists

  • How Sass represents values like, 10px 15px 0 0 or Helvetica, Arial, sans-serif.
  • Can be useful in some mixins, but maps are definitely the new hotness.
  • Indexes start at 1, not 0.
  • Can use "" or spaces

.scss

$cart-selectors: “.checkout”, “.remove”, “.add”;

#{$cart-selectors} {
  property: value;
}

.css

.checkout, .remove, .add {
  property: value;
}

List Functions - nth()

  • Get the nth item in a list.
  • nth($list, n)

.scss

$spacing: 5px, 10px, 20px, 30px;

div {
  margin: nth($spacing, 2);
  padding: nth($spacing, 3);
}

.css

div {
  margin: 10px;
  padding: 20px;
}

List Functions - join()

  • Join two lists together.
  • join($list1, $list2)

.scss

$list1: some strings;
$list2: with other strings;

join($list1, $list2) => some strings with other strings

List Functions - append()

  • Append a single value onto the end of a list.
  • Can choose comma or space as a separator
  • append($list, $value, $separator)

.scss

join(10px 20px, 30px) => 10px 20px 30px
join(10px 20px, 30px, comma) => 10px, 20px, 30px
join((red, blue), green, space) => red blue green

Maps

  • Maps represent an association between keys and values.
  • Keys are used to look up values.
  • Easy to collect values into named groups and access the groups dynamically.
  • They have no direct parallel in CSS.

.scss

$map: (key1: value1, key2: value2, key3: value3);
  • Must always be surrounded by parentheses.
  • Must always be separated by commas.
  • May only have one value associated with a given key (value may be a list)

Maps Function – map-get()

  • Returns the value in a map associated with a given key.
  • map-get($map, $key)

.scss

$map: (key1: value1, key2: value2, key3: value3);

map-get($map, key1) => value1

Maps Function – map-merge()

  • Merges two maps together into a new map.
  • map-merge($map1, $map2)

.scss

$map1: (key1: value1, key2: value2);
$map2: (key3: value3, key4: value4);

map-merge($map1, $map2) => $new_map: (key1: value1, key2: value2, key3: value3, key4: value4);

Maps Function – map-remove()

  • Returns a new map with keys removed.
  • map-remove($map, $keys...)

.scss

$map: (key1: value1, key2: value2, key3: value3, key4: value4);

map-remove($map, key1, key2) => $new_map: (key3: value3, key4: value4);

Maps Function – map-keys()

  • Returns a list of all keys in a map.
  • map-keys($map)

.scss

$map: (key1: value1, key2: value2, key3: value3, key4: value4);

map-keys($map) => key1, key2, key3, key4

Maps Function – map-values()

  • Returns a list of all values in a map.
  • map-values($map)

.scss

$map: (key1: value1, key2: value2, key3: value3, key4: value4);

map-values($map) => value1, value2, value3, value4

Loops - @if / @else

  • Conditionally apply styles based on the expression used.

.scss

$type: monster;
p {
  @if $type == ocean {
    color: blue;
  } @else if $type == matador {
    color: red;
  } @else if $type == monster {
    color: green;
  } @else {
    color: black;
  }
}

.css

p {
  color: green;
}

Loops - @for

  • Repeatedly outputs a set of styles based on a counter that increments between each output.
  • Has two forms:
    • @for $var from <start> through <end>
    • Includes everything up to the <end> value in the output, including the <end> value.
    • @for $var from <start> to <end>
    • Includes everything up to the <end> value in the output, including the <end> value.

.scss

@for $i from 1 through 3 {
  .item-#{$i} { width: 2em * $i; }
}

.css

.item-1 {
  width: 2em;
}
.item-2 {
  width: 4em;
}
.item-3 {
  width: 6em;
}

Loops - @each

  • The @each rule sets $var to each item in the list or map, then outputs the styles it contains using that value of $var.
  • Usually has the form @each $var in [$list or $map].
  • $var can be any variable name.

.scss

@each $animal in puma, sea-slug, egret, salamander {
  .#{$animal}-icon {
    background-image: url('/images/#{$animal}.png');
  }
}

.css

.puma-icon {
  background-image: url('/images/puma.png');
}
.sea-slug-icon {
  background-image: url('/images/sea-slug.png');
}
.egret-icon {
  background-image: url('/images/egret.png');
}
.salamander-icon {
  background-image: url('/images/salamander.png');
}

File Organization

Partials

  • Split your Sass files into smaller chunks to import into a manifest file.
  • Keep things as small as you need them to be to make sense for your project.
  • Use underscores when naming partials.
  • Common method for splitting up Sass files in a project:
stylesheets/
|
|-- modules/              # Common modules
|   |-- _all.scss         # Include to get all modules
|   |-- _utility.scss     # Module name
|   |-- _colors.scss      # Etc...
|   ...
|
|-- partials/             # Partials
|   |-- _base.sass        # imports for all mixins + global project variables
|   |-- _buttons.scss     # buttons
|   |-- _figures.scss     # figures
|   |-- _grids.scss       # grids
|   |-- _typography.scss  # typography
|   |-- _reset.scss       # reset
|   ...
|
|-- vendor/               # CSS or Sass from other projects
|   |-- _colorpicker.scss
|   |-- _jquery.ui.core.scss
|   ...
|
`-- main.scss             # primary Sass file

Imports – @import

  • What do you do with all the partial files you’ve created?
  • The very last file in the previous example was main.scss.
    • This is where you @import everything into a single manifest file.
    • You don’t need the underscores when using @import
    • You also don’t need to specify the filetype (.scss)
// Modules and Variables
@import "modules/all";

// Partials
@import "partials/base";
@import "partials/buttons";
@import "partials/figures";
@import "partials/grids";
@import "partials/typography";
@import "partials/reset";

// Third-party
@import "vendor/colorpicker";
@import "vendor/jquery.ui.core";
